## 🐛 Common Issues and Fixes

### ❌ Script executes but GUI doesn't appear
**Fix:** Press `RightShift` or `Insert` to manually toggle the GUI visibility. If still not showing, disable Discord overlay, GeForce Experience overlay, or any active ReShade filters — then re-inject and execute.

### ❌ Auto Rob getting caught / failing mid-heist
**Fix:** A game update likely changed guard patrol patterns or escape zone positions. Update to the latest script version — Auto Rob routing is the first module patched after every Mad City map update.

### ❌ Auto Arrest not triggering on criminals
**Fix:** Ensure you're in the Police role before activating Auto Arrest. The module performs a role check on activation — switching roles without toggling the module off and on again causes the targeting to fail.

### ❌ All Cars not showing unlocked vehicles
**Fix:** The All Cars unlock is session-based. If vehicles aren't appearing in the spawn menu, toggle the module off and on to reinitialize the catalog injection. Also confirm you're on the latest script version after any Mad City vehicle system updates.

### ❌ Script not working on Android via Delta
**Fix:** Confirm you're using **Delta v2.3+** and that Roblox was launched through Delta's built-in launcher. The **mad city script delta** loader requires the game to be fully loaded into the server before injection — wait for the spawn screen before executing.

### ❌ Silent Aim not registering hits
**Fix:** Reduce your FOV radius in the Silent Aim settings — if set too wide, the targeting may lock to players behind cover whose hit detection is server-blocked. A narrower FOV with medium smoothness gives the most reliable hit registration.

### ❌ "attempt to index nil value" error on execution
**Fix:** A recent Mad City update broke a variable reference. Check this repository's issues tab for a hotfix announcement, or wait for the next scheduled patch — typically available within a few hours of a Mad City update.
